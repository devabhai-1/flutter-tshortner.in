Step 1: Create .env File
-------------------------
1. In the react-app folder, copy .env.example to .env:
   
   Windows PowerShell:
   Copy-Item .env.example .env
   
   Mac/Linux:
   cp .env.example .env

2. The .env file is in .gitignore and will NEVER be committed to git
   (This keeps your Firebase config private!)

Step 2: Get Your Firebase Config
---------------------------------
1. Go to: https://console.firebase.google.com/
2. Select/Create your project
3. Click ‚öôÔ∏è (Project Settings)
4. Scroll to "Your apps" ‚Üí Web app
5. Copy each config value

Step 3: Fill in .env File
--------------------------
Open the .env file and replace with YOUR actual values:

VITE_FIREBASE_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXX
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_DATABASE_URL=https://your-project-default-rtdb.firebaseio.com
VITE_FIREBASE_PROJECT_ID=your-project-id
VITE_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=123456789012
VITE_FIREBASE_APP_ID=1:123456789012:web:xxxxxxxxxxxxx

‚ö†Ô∏è IMPORTANT: 
- No quotes needed around values
- No spaces around the = sign
- Must include databaseURL (required for Realtime Database)

Step 4: Enable Firebase Services
---------------------------------
In Firebase Console:

‚úÖ Authentication ‚Üí Sign-in method:
   - Enable "Email/Password"
   - Enable "Google"

‚úÖ Realtime Database ‚Üí Create Database:
   - Start in test mode (or use rules from README)
   - Note the database URL (you need this!)

Step 5: Install & Run
---------------------
cd react-app
npm install
npm run dev

That's it! App will open at http://localhost:5173

COMMON ERRORS & FIXES:
----------------------

‚ùå "Missing Firebase environment variables"
   FIX: Create .env file from .env.example and fill in all values

‚ùå "Cannot parse Firebase url"
   FIX: Add VITE_FIREBASE_DATABASE_URL in .env (must end with .firebaseio.com)

‚ùå "Permission denied"
   FIX: Check Realtime Database rules (see README.md)

‚ùå Changes to .env not working
